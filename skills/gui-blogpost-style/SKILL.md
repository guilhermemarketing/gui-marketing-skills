# Gui Blogpost Style — Guia de Estilo para Artigos Longos

Este documento codifica o estilo de escrita de Guilherme Lacerda para blogposts, extraído da análise de 5 textos autorais: "Campo Magnético de Vendas", "Full Stack Marketing e T-Shaped Marketer", "Copywriting Técnicas: Facas Ginsu", e os megathreads do Reddit (Aprendendo MKT Digital Gratuitamente + Plano de Carreira).

---

## 1. ESTRUTURA MACRO DO ARTIGO

### 1.1 Abertura — O Gancho Provocador
- Abre com uma afirmação ousada, uma crença popular sendo desafiada, ou uma pergunta retórica que cria tensão intelectual.
- Nunca abre com definição de dicionário ou introdução genérica ("Neste artigo vamos falar sobre...").
- Frequentemente usa itálico na primeira frase como recurso de voz interna ou citação cultural.
- O primeiro parágrafo deve causar uma micro-ruptura cognitiva no leitor.

**Padrões reais:**
- *"Primeiro a pessoa vê, depois ela se interessa, aí ela deseja, aí ela compra"… Sabe qual é o único problema com essa lógica?*
- *O mercado de marketing digital mudou radicalmente. Não basta mais saber apertar botões.*
- *Quem aqui se lembra de acordar de madrugada nos anos 90 e ser hipnotizado por uma propaganda frenética na TV?*

### 1.2 Corpo — Estrutura em Blocos Numerados
- Usa seções numeradas (1, 2, 3…) como capítulos, não bullets aleatórios.
- Cada seção tem um título descritivo e acionável.
- Dentro de cada seção: abertura contextual → argumentação com dados/exemplos → fechamento com ação prática.
- Parágrafos curtos (1-3 frases). Raramente ultrapassa 4 linhas por parágrafo.
- Alterna entre prosa explicativa e elementos visuais (listas, exemplos, destaques em negrito).

### 1.3 Fechamento — Conclusão com Manifesto
- A conclusão nunca é um resumo burocrático. É um manifesto compacto.
- Reforça a tese central com tom assertivo e inspirador.
- Frequentemente usa uma analogia final (ex: "Marketing é tipo medicina").
- Termina com uma frase-sentença curta e impactante, quase como um slogan.
- Pode incluir CTA para a comunidade, newsletter ou produto, mas de forma orgânica.

**Padrão real:** *"Esse é o caminho."* / *"This Is The Way."* / *"Cabe a você decidir se vai ser mais um apertador de botões ou um estrategista completo."*

## 2. TOM DE VOZ E CADÊNCIA

### 2.1 O Tom — Provocador Generoso
- Provocativo mas nunca arrogante. Desafia o leitor mas sempre oferece a saída.
- Mistura autoridade técnica com acessibilidade coloquial.
- Usa "você" diretamente — conversa com o leitor como se fosse um mentor na mesa de bar.
- Usa expressões coloquiais brasileiras naturais: "vira só um apertador de botão", "rasgar dinheiro", "calma aí, campeão", "uma porrada de ferramentas".
- NUNCA usa linguagem corporativa vazia ("sinergia", "alavancagem holística", "paradigma disruptivo").

### 2.2 Cadência — Ritmo Cinematográfico
- Alterna frases longas (explicativas) com frases curtas (impactantes).
- Usa frases isoladas como parágrafos para criar punch:

```
Sem controle. Só atração.
```

```
Esse é o caminho.
```

```
É método + dados + leitura de contexto + ajustes constantes.
```

- Usa a técnica de "parágrafo-bomba": uma frase curta sozinha após um bloco argumentativo longo para criar impacto máximo.

### 2.3 Recursos Retóricos Recorrentes
- **Analogias do cotidiano:** "Marketing é tipo academia: se você não for consistente, nunca verá resultados." / "Marketing é tipo medicina."
- **Perguntas retóricas intercaladas:** "De que adianta ser o primeiro no Google se a sua página não converte?"
- **Citações emolduradas em blockquote:** Usa `>` para frases-chave que sintetizam o argumento da seção.
- **Negrito estratégico:** Negrita conceitos-chave e frases de impacto, nunca parágrafos inteiros. O negrito funciona como semáforo visual.
- **Itálico para voz interna ou termos técnicos ingleses:** *"feeling"*, *"stacking"*, *"profit-first"*.
- **Parênteses explicativos:** Usa parênteses para traduzir siglas ou dar contexto rápido: "CAC (Custo de Aquisição de Cliente)".

## 3. PROFUNDIDADE ARGUMENTATIVA

### 3.1 Fundamentação — Sempre com Lastro
- Nunca faz afirmação solta. Fundamenta com: dados percentuais, cases (anônimos ou próprios), frameworks nomeados (SPIN, AIDA, PAS, FAB), ou referências teóricas (Kahneman, Schwartz, Hopkins, Gladwell).
- Quando cita um framework, explica brevemente E mostra aplicação prática imediata.
- Não academicismo vazio — a teoria serve à prática, não o contrário.

### 3.2 Exemplos Práticos — O Coração do Conteúdo
- Cada conceito tem pelo menos um exemplo prático específico (nunca genérico).
- Os exemplos usam cenários reais: "Se você vende consultoria fitness...", "Se você é contador...", "Um cliente gastava R$ 30k/mês...".
- Usa o formato "Exemplo prático:" em negrito como separador antes do exemplo.
- Frequentemente usa "Ação Imediata:" como CTAs internos ao longo do texto (não apenas no final).

### 3.3 Cross-linking Intencional
- Referencia outros conteúdos do ecossistema gui.marketing de forma contextual.
- Links internos são inseridos naturalmente na argumentação, nunca como lista genérica de "leia também".
- Menciona metodologias próprias (Brandformance, Funil Invertido, Campo Magnético) como conceitos vivos, não autopromoção.

## 4. FORMATAÇÃO VISUAL

### 4.1 Hierarquia de Headings
- H1: Título do artigo (uma vez)
- H2: Seções principais (numeradas ou temáticas)
- H3: Subseções dentro de cada bloco
- Não ultrapassa H3 em profundidade. Prefere criar novos H2 a aninhar demais.

### 4.2 Listas e Bullets
- Usa bullets quando está listando elementos paralelos (canais, ferramentas, métricas).
- Bullets são curtos (1 linha, máximo 2) com conceito em negrito + explicação.
- Para processos sequenciais, usa listas numeradas.
- Evita listas com mais de 7 items (quebra em subcategorias se necessário).

### 4.3 Elementos Visuais de Texto
- **Blockquotes (>):** Para frases-síntese que resumem a tese da seção.
- **Separadores (---):** Entre grandes blocos temáticos, não entre cada parágrafo.
- **Sumário/TOC:** Artigos longos incluem sumário clicável no início.
- **Imagens e mapas mentais:** Referencia imagens/diagramas quando relevante, com legendas descritivas.

## 5. VOCABULÁRIO E EXPRESSÕES ASSINATURA

### 5.1 Expressões Recorrentes (usar naturalmente, não forçar)
- "apertador de botão/botões" (para profissionais sem visão estratégica)
- "rasgar dinheiro" (para desperdício de verba)
- "métrica de vaidade" (para dados superficiais)
- "ciência pura" (para referir-se a marketing baseado em dados)
- "fórmula mágica" / "receita de bolo" (para o que ele combate)
- "o jogo muda" (para momentos de virada)
- "papo-reto" / "a real é simples" (para verdades diretas)
- "campeão" (endereçamento casual ao leitor)
- "This Is The Way" (fechamento-assinatura)

### 5.2 Termos Técnicos — Sempre Traduzidos
- Ao usar sigla inglesa, traduz na primeira menção: "CAC (Custo de Aquisição de Cliente)".
- Usa os termos técnicos reais (não simplifica para público leigo demais).
- Equilibra jargão técnico com explicações acessíveis.

### 5.3 O que NUNCA escrever
- "Neste artigo, vamos explorar..." (abertura genérica)
- "Em conclusão..." / "Para resumir..." (fechamento burocrático)
- Qualquer frase que comece com "É importante ressaltar que..."
- Parágrafos com mais de 5 frases
- Tom neutro/passivo sem posicionamento claro
- Elogios a gurus ou menções positivas a "hacks rápidos"
- Linguagem de ChatGPT genérico ("revolucionário", "game-changer", "poderoso")

---

## 6. REGRAS DE OURO

1. **Provocação com solução.** Toda provocação no texto deve ter uma resolução prática. Nunca critique sem mostrar o caminho.
2. **Exemplos > Teoria.** Na dúvida entre mais teoria ou mais exemplo, sempre escolha o exemplo.
3. **Parágrafos curtos.** Se um parágrafo tem mais de 4 linhas, quebre.
4. **Uma ideia por seção.** Cada H2/H3 deve ter um conceito central claro.
5. **Blockquote = Frase que o leitor vai printar.** Use com sabedoria, máximo 3-4 por artigo.
6. **Cross-link com contexto.** Só linke se a referência enriquece o argumento atual.
7. **CTA interno ao longo do texto.** Não espere o final. Use "Ação Imediata:" e CTAs para materiais em pontos estratégicos.
8. **Fechamento-manifesto.** A conclusão é uma declaração de princípios, não um resumo.
9. **Negrito é semáforo.** Use para guiar o scanner visual. Não abuse.
10. **O texto deve soar como mentoria, não como palestra.** Tom de conversa 1:1, não de TED Talk.

---

## 7. CHECKLIST PRÉ-PUBLICAÇÃO

Antes de entregar o blogpost, verifique:

- [ ] A abertura causa uma ruptura ou tensão? (não é genérica?)
- [ ] Cada seção tem pelo menos um exemplo prático específico?
- [ ] Há blockquotes com frases-síntese "printáveis"?
- [ ] Os parágrafos têm no máximo 3-4 linhas?
- [ ] Termos técnicos estão traduzidos na primeira menção?
- [ ] Há CTAs internos (Ação Imediata / links para ferramentas)?
- [ ] O fechamento é um manifesto, não um resumo?
- [ ] O tom soa como mentoria provocadora, não palestra acadêmica?
- [ ] Nenhuma expressão genérica de "IA assistant" escapou?
- [ ] O texto faz o leitor querer compartilhar pelo menos 1 frase?
